Let's talk Node

Did we even need another programming language?

  • Brings event-driven programming to web servers - no need for threading
  • Lightweight
  • Stateless
  • Non-blocking IO
  • Perfect for microservices
  • NPM
- Respond to system events, nothing on its own
- APIs rule - you get functionality from it
- 20-30Mb of RAM while idle, no compiling
- The only source of truth should be some external db
- Every operation on system is async
- Apps nowadays - API consumers, API glue
- highly scalable (stateless apps), performant
- you want to make it, and you want to make it in a day
